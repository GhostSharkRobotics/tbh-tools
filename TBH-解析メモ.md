# TBH 解析メモ（ゲーム本体抽出で分かった「見えない情報」）

ゲームビルド v1.00.09 を抽出して判明した、ゲーム内では見えない仕様の記録。
データの一次ソースは `tools/extracted/tbh-db.json`（45テーブル）。ツール用は `tbh-data.json`。

## 1. 合成（キューブ Synthesis）

**仕様**：同じグレード × 同じカテゴリの装備 **9個** → より高いグレードの装備 **1個**。

- **カテゴリは3つ・跨げない**：`Gear`(武器+防具) / `Accessory`(アクセ) / `Material`(宝石・彫刻・刻印・クラフト素材など)
- **結果は入れた種類に無関係＝カテゴリ固定プールから重み付きランダム**
  - 武器だけ9個入れても **防具が出る**（兜・鎧・手袋・靴）
  - 宝石だけ入れても **彫刻やクラフト素材が出る**
  - 狙った種類は指定できない
- **昇格確率**（GradeInfoData の重み）：
  - コモン〜レジェンダリー：**ほぼ100%で +1**（5%前後で +2 ジャンプ）
  - **イモータルから「据え置き」発生**：イモータル50% / アルカナ67% / ビヨンド77% / セレスティアル83% / ディバイン91% が同グレードのまま
  - コズミックは合成不可（最上位）
  - 降格は無い（Lower重み=全0）
- **解放条件**：イモータル合成＝キューブLv10、セレスティアル合成＝キューブLv50 から
- ゲームは合成前に「上位等級の確率：◯%／同等級：◯%」「結果レベル範囲：◯〜◯」を表示する

### ★重要：産出装備ごとの「重み」は別テーブルにある（見落とし注意）
- `SynthesisRecipeInfoData`（レシピ定義）には **weight も産出itemリストも無い** → ここだけ見ると「重み無し」と誤結論する
- **重みは `SynthesisDropInfoData`→`DropKey`→`DropInfoData[DropKey].Weight` にある**（職限定の追加分は HeroKeyCondition=501/601）
- ※注意：DropInfoData の RewardKey は**ItemGroup（クラス別武器をまとめた束）**を指す。例: group 1100010 = 剣+弓+杖+セプター。だから1グループの重みは複数種に配分される（種類別%を出すときはグループ展開が必要）。

合成と違い、クラフトは**作る部位タイプを選ぶ**ので「何が出る」が部位で決まる。`CraftingRecipeInfoData`（56件）も各レシピに `DropKey` を持ち、`DropInfoData` に重みあり。

## 3. ★DLC条件（職501/601）の正体 ＝ パーティ編成ではなく「DLC所有」

ドロップの `HeroKeyCondition`（職コード）は **そのヒーローのDLCを所有しているか** を指す。現在の編成とは無関係。

- ヒーローのDLC状況（HeroInfoData）：
  - 101 ナイト / 201 レンジャー / 301 ソーサラー：**無料**（DLCAppId=0）
  - 401 プリースト：DLCあり（DLCAppId=4427410）だが **DLC専用ドロップは無し**（HasDLCDrop=False）
  - **501 ハンター：DLC（4427420）・`HasDLCDrop=True`**
  - **601 スレイヤー：DLC（4427430）・`HasDLCDrop=True`**
- 職条件付きドロップは 501（977件）/ 601（977件）に集中。DropType=`EachDropOneWeight_DLCVariant`
- **意味**：Hunter DLCを持つと結果プールに **クロスボウ/ボルト** が追加、Slayer DLCを持つと **斧/ハチェット** が追加される
- **→ 合成/クラフト/箱の結果%は「所有しているDLCで変わる」**。精度を出すなら「DLC無し / +Hunter / +Slayer」で別表示すべき

## 4. 箱（チェスト）システム

- 白箱(910/COMMON)＝通常敵 / 青箱(920/RARE)＝**ステージボス** / 金箱(930/LEGENDARY)＝章ボス
- ドロップ率は**ステージ固定**（白箱8〜16%、青箱はボスから80〜100%）、中身は重み付きテーブル（職DLC条件あり）
- 「白箱を満杯にすると青箱が出やすい」は**データに該当ロジック無し**（ドロップ率は在庫状態に非依存）＝迷信。青箱を増やす実手段は「ステージボスを倒す」＋進行ツリー `DropChanceStageBossChestPercent`

## 5. Steam市場の絞り（2026更新）

- 現行の制限＝**取引可否**：`IsCanExchangeMarketable=True` は820件のみ（Legendary+のAロール・特定レベルだけ）。Bロール・下位・非対象レベルは取引不可（ゲーム内には存在）
- 「時間あたりドロップ上限」の仕組み（Steam在庫 `drop_window`/`max_per_window`）は**フィールドはあるがこのビルドでは全て0=無効**。あるとすればサーバー側でクライアントデータには出ない

## 6. その他の確定事項

- **武器ベース攻撃速度**＝GearInfoData BaseStat2（レアリティで上昇、`tbh-data.json` の equipment に `base`+`/s` で収録済み）。例：弓 base÷100 ＋ Rangerクラス基礎1.0 ＝ コモン弓1.3〜1.4 / コズミックLv65弓4.10
- **武器DPSの本質は攻速差**：Lv90コズミック基礎で弓3.1/s vs スタッフ0.4/s → 弓の素DPSが約5倍
- **固有効果（Arcana+）**：ゲーム公式の説明テキスト（数値入り・16言語）が `UniqueMod_<key>` にあり、実param値も収録済み
- **被ダメージ**：軽減前は算出可能（敵ATK×スキル倍率×ステージLv補正×ボス補正）。軽減式はゲームコード内（IL2CPP・難読化）でデータには無い

## 教訓（情報が失われないために）
- 「○○テーブルに無い＝存在しない」は危険。`DropKey` 等のKey参照で繋がる**別テーブルを辿る**（probonk/CSVは正規化されていて重み・産出物は別表に分離）
- DLC条件は結果分布を変える。DLC所有を前提に置かないと%がズレる
